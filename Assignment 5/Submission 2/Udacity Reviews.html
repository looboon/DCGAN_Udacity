Build the Neural Network

The function model_inputs is implemented correctly.

Good work. You correctly implemented the placeholders while ensuring the datatype was correct!

The function discriminator is implemented correctly.

Excellent work on implementing multiple conv layers and appropriately applying the necessary activation functions and using batch normalization.

Couple of suggestions -

  • Think about how filter size affects your model learning features for a CNN. As you increase the layers, your model learns more features, so does a larger filter size make sense or a smaller one to learn more features better?
  • You currently use flat = tf.reshape(drop3, (-1, 4*4*256)) for the Dense layer. Alternatively, you could simplify this step by using flat = tf.contrib.layers.flatten(drop3). Slightly more convenient (I hate trying to reshape things :D )
  • Do consider creating a separate function for leaky ReLU here. It's good programming practice.

Also, GANs are difficult to optimize. If the margin by which the discriminator wins is too big, then the generator can’t learn well as the discriminator error would be too small. If the generator wins by too much, it won't learn well because the discriminator is too weak to teach it. There is a lot of research in GANs as you might know to solve all kinds of such problems. There are several recommended ways, some which you already tried based on the previous review.

  • Definitely, try out weight initialization. Some research papers recommend Xavier initialization as mentioned before. https://www.tensorflow.org/api_docs/python/tf/contrib/layers/xavier_initializer
  • I believe this has been referred previously, but do try to read on this - https://arxiv.org/pdf/1606.03498v1.pdf

Important

You have added dropout here, which is great. But I would like to ask you this important question - dropout should not be added during inference, that's why the keep_probability value is set to 1.0 for inference. You added dropout here, but what about the keep_prob for inference? Do you need to modify your code somewhere else for that? Think about this and how it could be affecting your model if at all.

The function generator is implemented correctly.

Nicely done!

One small suggestion -

  • For your logits, you use tf.layers.conv2d_transpose() which has an argument activation. So to create out you can directly pass tf.nn.tanh as a parameter to conv2d_transpose. It doesn't make too much of a difference except making it a bit more concise.

Important

I understand the previous reviewer asked you to have a larger model for generator. While that is good advice, please remember that you are working with limited epochs for this project. For a larger model, you would require more epochs to be able to train it well. Since you are restrained by the number of epochs, a relatively smaller model should do as well :)
